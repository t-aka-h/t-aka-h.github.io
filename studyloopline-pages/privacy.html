Studyloopline — Privacy Policy

Last updated: 2026-06-03 (v1.0 — initial release)

English 日本語

Studyloopline ("the App") is developed and operated by an independent developer. This Privacy Policy describes how the App handles user information.

Summary

  • The App does not collect personal data on developer servers.
  • The App does not contain analytics SDKs.
  • The App does not display advertisements.
  • The App does not contain in-app purchases.
  • All learning progress is stored locally on your device using iOS UserDefaults / SwiftData.

Data Collection

The App does not collect any personal data on developer servers. Local learning state — practice answer history, Level Check results, accuracy by topic, learning streak, theme preference — is stored on your device using iOS standard storage (UserDefaults and SwiftData). This data never leaves your device.

Camera (QR Code Scanning)

Studyloopline does not request camera access. The App itself never accesses the camera.

Local Storage

The App stores the following data locally on your device. This data never leaves your device unless you explicitly enable the Personal Backend feature.

  • Practice answer history (which problems you've solved, correct/incorrect, timestamp)
  • Level Check results (which level you were assigned, accuracy)
  • Theme preference (Sky / Ocean / Sunset / Forest / Lavender / Mono)
  • UI preferences (Japanese / English language follows iOS system setting)

You can delete this data at any time by uninstalling the App.

Third Parties

Studyloopline does not integrate with any third-party SDKs, analytics services, advertising networks, or data brokers. The App uses only Apple's first-party frameworks:

  • SwiftUI — UI framework
  • SwiftData — local data persistence
  • AVFoundation — audio playback (only when Personal Backend is configured)
  • StoreKit — version checking (no purchases)

Children

The App is suitable for all ages (Apple Age Rating 4+). It does not knowingly collect data from children.

Personal Backend (Optional)

This feature is strictly optional and is not enabled by default. Most users will never use it, and the App is fully functional without it (the Lab modules, Level Check, the 177 Practice problems, and the Calculator all work entirely offline).

Studyloopline includes an optional advanced feature that lets you connect the App to a personal backend that you run yourself — typically a Cloudflare Worker + R2 bucket on your own account, executing an open-source learning loop (Claude Code + NotebookLM, etc). This feature exists for users who want to build their own personalized statistics tutor; it is not a service we provide.

This feature is hidden by default and must be unlocked explicitly by the user. Unless you take explicit action to enable and configure it, the App makes no outbound network requests for learning content. When (and only when) enabled and configured, the App communicates exclusively with the server URL you specify, using a token you control:

  • The App fetches today's questions and audio mini-lecture from your server.
  • The App submits your answers to your server (so your backend can grade and adapt to your level).
  • The App fetches past lectures, feedback messages, and a personal changelog from your server.

The developer of Studyloopline has no access to data exchanged between your device and your personal backend. Your backend is yours; your data stays in your own account. The developer never sees your server URL or your token, and operates no shared backend for this feature.

You can disable this feature at any time from the App's settings, which clears the saved server URL and token from your device.

When this feature is configured, the App can stream audio mini-lectures from your own server using Apple's AVFoundation (playback category, to allow background playback). No audio recording occurs.

Changes to this Policy

If we change this Privacy Policy, we will update the "Last updated" date above. Material changes will be communicated through the App's release notes.

Contact

If you have any questions about this Privacy Policy, please contact us here.

Studyloopline — プライバシーポリシー

Studyloopline（以下「本アプリ」）は、個人開発者によって開発・運営されています。本ポリシーは、本アプリにおけるユーザー情報の取扱いを説明します。

サマリー

  • 本アプリは開発者サーバーに個人情報を送信しません。
  • 本アプリは分析 SDK を含みません。
  • 本アプリは広告を表示しません。
  • 本アプリはアプリ内課金を含みません。
  • 学習進捗はすべてデバイス上に UserDefaults / SwiftData でローカル保存されます。

データ収集

本アプリは開発者サーバーに個人情報を一切収集しません。学習状態（練習問題の回答履歴・レベル判定結果・トピック別正答率・連続学習日数・テーマ設定）は、iOS 標準ストレージ（UserDefaults および SwiftData）でデバイス上にローカル保存されます。これらのデータがデバイスを離れることはありません。

カメラ

本アプリはカメラへのアクセスを要求しません。本アプリがカメラにアクセスすることはありません。

ローカルストレージ

本アプリは以下のデータをデバイス上にローカル保存します。パーソナルバックエンド機能を明示的に有効化しない限り、これらのデータがデバイスを離れることはありません。

  • 練習問題の回答履歴（解いた問題・正誤・タイムスタンプ）
  • レベル判定結果（判定されたレベル・正答率）
  • テーマ設定（Sky / Ocean / Sunset / Forest / Lavender / Mono）
  • UI 設定（日本語 / 英語は iOS のシステム設定に従います）

これらのデータは本アプリをアンインストールするだけで完全に削除されます。

第三者サービス

Studyloopline は第三者 SDK・分析サービス・広告ネットワーク・データブローカーと一切連携していません。本アプリは Apple の純正フレームワークのみを利用しています:

  • SwiftUI — UI フレームワーク
  • SwiftData — ローカルデータ永続化
  • AVFoundation — 音声再生（パーソナルバックエンド設定時のみ）
  • StoreKit — バージョン確認（課金はありません）

子ども

本アプリは全年齢対象（Apple Age Rating 4+）です。子どもからの情報収集は一切行いません。

パーソナルバックエンド（任意）

本機能は完全に任意で、初期状態では無効です。ほとんどのユーザーはこの機能を使う必要はなく、本アプリは本機能を使わなくても完全に動作します（実験室・レベル判定・177 問の練習・電卓はすべてオフラインで利用できます）。

本アプリには、あなた自身が運営するパーソナルバックエンドに接続できる任意の上級機能があります。バックエンドは通常、あなた自身のアカウントで動かす Cloudflare Worker + R2 バケットで、オープンソースの学習ループ（Claude Code + NotebookLM など）を実行します。この機能は、自分専用のパーソナル統計チューターを構築したいユーザー向けに存在するものであり、開発者が提供するサービスではありません。

本機能は初期状態では隠されており、ユーザー自身が明示的に有効化する必要があります。あなたが明示的に有効化・設定しない限り、本アプリは学習コンテンツに関する外部通信を一切行いません。有効化・設定した場合に限り、本アプリは あなたが指定したサーバー URL とのみ、あなたが管理するトークンを使って通信します:

  • 本アプリは、あなたのサーバーから今日の問題と音声ミニ講義を取得します。
  • 本アプリは、あなたのサーバーに回答を送信します（あなたのバックエンドが採点とパーソナライゼーションを行えるように）。
  • 本アプリは、あなたのサーバーから過去の講義・フィードバック・変更履歴を取得します。

Studyloopline の開発者は、あなたのデバイスとあなたのパーソナルバックエンドの間でやり取りされるデータには一切アクセスできません。バックエンドはあなたのもので、データはあなた自身のアカウントに留まります。開発者があなたのサーバー URL やトークンを見ることはなく、本機能のために開発者が共通バックエンドを運営することもありません。

本機能は、アプリの設定画面からいつでも無効化できます。無効化するとデバイスに保存されたサーバー URL とトークンは消去されます。

本機能を設定した場合、Apple の AVFoundation を用いてあなた自身のサーバーから音声ミニ講義をストリーミング再生できます（バックグラウンド再生のため playback カテゴリ）。録音は一切行いません。
